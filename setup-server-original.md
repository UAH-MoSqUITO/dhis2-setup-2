# Server setup

These are the original steps from (approximately, 2.31.1) [9.4 Server setup](https://docs.dhis2.org/master/en/implementer/html/install_server_setup.html), along with corrected commands and execution logs.

1. Creating a user to run DHIS2

    _Original_

    ```shell
    sudo useradd -d /home/dhis -m dhis -s /bin/false
    sudo passwd dhis
    ```

    _Log_

    ```console
    vagrant@vagrant:~$ sudo useradd -d /home/dhis -m dhis -s /bin/false
    vagrant@vagrant:~$ sudo passwd dhis
    Enter new UNIX password:
    Retype new UNIX password:
    passwd: password updated successfully
    ```

2. Creating the configuration directory

    _Original_

    ```shell
    mkdir /home/dhis/config
    chown dhis:dhis /home/dhis/config
    ```

    _Corrected_

    ```shell
    sudo mkdir /home/dhis/config
    sudo chown dhis:dhis /home/dhis/config
    ```

3. Setting server time zone and locale

    _Commentary_

    We can skip this section entirely, given our use case. The default timezone for Ubuntu 18.04 in Vagrant and in AWS EC2 is UTC, which matches the demo data: ["Sierra Leone observes Greenwich Mean Time all year. There are no Daylight Saving Time clock changes."](https://www.timeanddate.com/time/zone/sierra-leone). Also, US English is the default locale, at least when I spin up a new virtual machine on a host that is already set to use US English.

    _Original_

    ```shell
    sudo dpkg-reconfigure tzdata
    locale -a
    sudo locale-gen nb_NO.UTF-8
    ```

    _Corrected but not used_

    ```shell
    sudo dpkg-reconfigure tzdata
    locale -a
    sudo locale-gen en_US.UTF-8
    ```

4. PostgreSQL installation

    _Commentary_

    In Ubuntu 18.04, the `postgresql-contrib-10` package is "purely virtual", and seems to be part of the base `postgresql-10` package. That is, it can be omitted from the `apt-get install` arguments.

    Also, the `createuser` command will ask for a password, which is needed later in step XX. I used a randomly generated password.

    _Original_

    ```shell
    sudo apt-get install postgresql-10 postgresql-contrib-10 postgresql-10-postgis-2.4
    sudo -u postgres createuser -SDRP dhis
    sudo -u postgres createdb -O dhis dhis2
    sudo -u postgres psql -c "create extension postgis;" dhis
    ```

    The original instructions also include a couple of strange mentions (in paragraphs) of exiting, but they are inconsistent with the other commands.

    _Corrected_

    ```shell
    sudo apt-get install postgresql-10 postgresql-contrib-10 postgresql-10-postgis-2.4
    sudo -u postgres createuser -SDRP dhis
    sudo -u postgres createdb -O dhis dhis2
    sudo -u postgres psql -c "create extension postgis;" dhis2
    ```

    _Original_

    ```shell
    sudo -u dhis nano /home/dhis/config/dhis.conf
    chmod 0600 dhis.conf
    ```

    ```properties
    # Hibernate SQL dialect
    connection.dialect = org.hibernate.dialect.PostgreSQLDialect

    # JDBC driver class
    connection.driver_class = org.postgresql.Driver

    # Database connection URL
    connection.url = jdbc:postgresql:dhis2

    # Database username
    connection.username = dhis

    # Database password
    connection.password = xxxx

    # Database schema behavior, can be validate, update, create, create-drop
    connection.schema = update

    # Encryption password (sensitive)
    encryption.password = xxxx
    ```

7. Java installation

    _Commentary_

    The server setup documentation says that Oracle's Java 8 JDK is recommended, but Oracle wants companies to start paying for support for Java 8, starting Jan 2019. As of Sep 2018, the default Java for Ubuntu 18.04 has been openjdk 11, but packages for openjdk 8 are also available without changes to APT configuration.

    ```shell
    sudo add-apt-repository ppa:webupd8team/java
    sudo apt-get update
    sudo apt-get install oracle-java8-installer
    java -version
    sudo apt-get install oracle-java8-set-default
    ```

    ```shell
    sudo apt-get install openjdk-8-jdk
    ```
